# Формати даних

Дані, що ЗВО проходять декілька етапів обробки і можуть бути описані наступними рівнями:

## Рівень 0 (сирі дані)

- Записуються вручну в реєстраційні картки
- Містять час, тип вимірювання, покази з лімбів приладу, температуру, погодні умови

## Рівень 1 (цифрові сирі дані)

- Файли **vtx** — текстові блоки з інформацією про тип вимірювання, температуру, покази з лімбів, коментарі
- Зберігаються у вигляді щомісячних директорій

## Рівень 2 (оброблені дані)

- Файли **voz** — містять дату, час, тип вимірювання, покази з лімбів, обчислені значення ЗВО (в одиницях Добсона)
- Дані використовуються для побудови часових рядів, середньодобових значень, сезонних трендів

З переходом на оновлене програмне забезпечення створюються також файли формату `_.NNN` (для поточного приладу `_.123`), що містять безпосередні виміри з приладу, обрахований рівень ЗВО та калібрувальні таблиці.

## Umkehr виміри

Umkehr виміри, що містять вертикальний розподіл озону, на рівні 0 мають аналогічні сирі дані, як і для ЗВО, але завжди проводяться у смузі довжин хвиль **С** та в ранковий або вечірній час.

Обробка даних для Umkehr проводиться за допомогою спеціалізованого програмного забезпечення в напівавтоматичному режимі та потребує роботи кваліфікованого спеціаліста. Результуючий профіль розподілу концентрації озону з висотою записується в **URZ-файлі**, який складається з заголовку та набору вимірів.
